    What are layers

    The OSI models defines 7 different layers which form a full network stack.

    Layer Function
    Application The application layer is the final destination for data. It is what uses the data (ie a web browser).
    Presentation This layer converts data between data friendly for transportation and data friendly for the end application. An such a conversion would be encryption.
    Session When a client and server connect they establish a session and data related to this (such as timing tolerances) is handled here.
    Transport This layer handles how data should be sent, ie size of packets, fault tolerance etc. This is where TCP/UDP is.
    Network The network layer handles the underlying functionality of how packets get to their destination, ie routing and forwarding packets
    Data Link The link layer handles the conversion from digital data to a form friendly for the transport medium as well as providing error correction for the physical layer.
    Physical This is the final layer representing physical real world infrastructure such as the wiring or electrical connections.

    The TCI/IP model is the same except Application, Presentation and Session are all combined into Application. This is an example of how data flows through these layers using the TCP/IP model.

    Hardware: Data comes in from cable ⇢ Link: Converted to digital representation ⇢ Internet: Is the packet at the correct place ⇢ Transport: Fault check and merge packets ⇢ Application: Use the data for its intended purpose

    ARP (Address resolution protocol)

    As the link layer only has MAC addresses, we need a way to convert a MAC address into an IP address. This is what ARP does.

    A link layer broadcast asks connected devices if they know the IP for a given MAC address. Eventually (or not) a host replies informing which IP is associated with that MAC address. There are 4 different broadcast messages that can be sent:

    • Unicast: Sent to a single recipient
    • Broadcast: Sent to everyone connected
    • Multicast: Sent to anyone who is interested
    • Anycast: Sent to anyone (usually the closest), doesn't matter who as long as it reaches someone

    There are some vulneribiities in this protocol due to it's inherent trust of other hosts. Other hosts can spoof details and steal frames from others or, particularly low powered ones, can sleep and not respond to requests.
